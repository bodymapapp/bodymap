What changed, what stayed

This is a pure visual restyle, no content deletions. Every feature listed in the audit before this build is preserved. The only changes are the visual chrome (tab palette, header typography, card radii, spacing) to match BillingDashboardV2.

Chrome changes (the only changes)

Period tabs background #F3F4F6 cool gray #F5EFE0 cream-deep
Period tabs distribution Natural width flex: 1 equal width
Period tabs active text #1F2937 #1F4131 forest-deep
Period tabs count 4 tabs 5 tabs (Yearly added)
Page title font System sans Cormorant Garamond, 36px, weight 600

What stayed (verified against the audit)

Right pane: TimelineView

  • Vertical pixel-positional canvas, 0.85 px per minute, 8am to 7pm default
  • Auto-stretches if appointments fall outside the default window
  • 48px left gutter for hour labels
  • Horizontal hairlines per hour
  • Date nav header with Prev / Today label / Next + "Back to Today" link when offset isn't 0
  • 11-day horizontal day picker scroll row (-7 to +3), past days dimmed, today highlighted, auto-scrolls to today
  • Collapsible Legend pill with 5 swatches (Brief ready, No intake yet, Complete, From Google, Preview)
  • Long-press 500ms anywhere on canvas to create a personal-time block (modal with editable start/end via InlineTimeInput, optional reason, "Block this time" or "Schedule a session at this time instead" buttons per Phase 9.3)
  • Long-press cancels on pointermove greater than 10px (so scroll doesn't trigger)
  • Therapist's own blocked time rendered as 45° amber stripes with label and reason
  • Gaps under 90 min: amber dashed stripes with "fill this gap" pill
  • Gaps over 90 min: soft amber gradient with "Open, Nh available" label
  • Now-line marker (red dot + bar) on today only, positioned by current minute
  • Empty-day hint centered with long-press instruction
  • Appointment cards positioned by minute, color-coded by status (intake-done green, pending-intake amber, complete gray, paid cream, preview dashed gray, Google purple)
  • Border-left 4px solid status color stripe on each card
  • Avatar circle, name, time, duration, reminder indicator, service, location pill
  • Status pill, paid pill, deposit-due pill, brief-ready pill conditionally rendered
  • Click appt to select, opens DetailPanel slide-over

Left rail: SmartBookingRail (every tab)

  • UpNextCarousel: 3-point briefing cards per upcoming client, ‹ › nav, dot indicators
  • BodyLoadCard: today's service mix vs injury risk meter, scope-aware
  • TimeAvailableCard (NEW, replaces RevenueCard): collapsed by default showing hours open + window count + tap-to-expand. Expanded reveals named windows + 3 industry-best-practice strategies, each with a single primary action button. Strategies adapt to time-tier (light, comfortable, spacious, fully-booked).
  • FillGapCard: one matched client per gap, one-tap text outreach (hidden when no real gap)
  • RebookWatchCard: top 3 lapsed regulars approaching cadence break (hidden when no candidates)
  • Body Load + TimeAvailable render 2-up on mobile, stacked on desktop (preserved layout)

Above the layout

  • App header (logo, brand, plan pill)
  • Page header: "Schedule" + date subtitle
  • Stats strip: 4 numbers with vertical separators (Today, Brief ready, Need intake, This week)
  • Action buttons: Book Appointment, Previews toggle, Time off
  • Period tabs (now 5 instead of 4)
