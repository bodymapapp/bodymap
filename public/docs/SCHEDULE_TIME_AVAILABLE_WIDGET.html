Design notes

The widget replaces the Revenue card in the schedule rail. Same physical slot, different framing: instead of measuring how much money was made, measure how much time is open and what to do with it.

Why this works

  • Time is the underlying asset for a solo LMT. Revenue is a downstream consequence. Framing time directly skips the stress of "you're behind."
  • Empty time is not loss. It's capacity. The widget treats it as raw material, not a problem to fix.
  • Strategies are options, not assignments. Each one is a single-button suggestion. Therapist takes it or ignores it. No guilt.
  • The fully-booked empty state celebrates instead of deflates. "Beautiful work" before any prompts.

Tier logic

Hours open this weekTierStrategy emphasis
0 Fully booked Waitlist · Rate review · Self-care block
0 to 5 Light Outreach · Social post · Review request
5 to 15 Comfortable Outreach · Cluster opportunity · Premium service slot
15 and over Spacious Wider outreach · Self-care · CEU/learning

Strategy library (industry best practices)

The widget draws from a fixed library of ~10 strategies, picking 3 per render based on tier and available context. Strategies match real practice patterns:

  • Outreach · always shown when RebookWatch has candidates. Wording adjusts for count.
  • Cluster opportunity · detects isolated single sessions on the same day, suggests grouping.
  • Premium service slot · when a 90-min+ window is available, suggests offering a higher-value service.
  • Social post / review request · light marketing actions for small windows.
  • Self-care block · always relevant when more than ~10h are open, or when 0h open as a counterbalance.
  • CEU / learning · for spacious weeks.
  • Waitlist setup · shown when fully booked (capacity question is real).
  • Rate review · shown when consistently fully booked (this needs cross-week data; v1 just shows it on a fully-booked week).

Scope

  • Today tab · today's open windows. Empty state: "Today is done. Check Weekly to see ahead."
  • Weekly tab · this week's open windows (default in the mockup above)
  • Monthly tab · this month's open windows, strategies tilt toward structural review
  • Yearly / Insights · widget either hidden or shows seasonal patterns instead

Implementation notes

  • The gap calculation already exists in TimelineView; lift the gaps-over-60-min logic into a shared helper.
  • Bookable window detection should respect blocked time and the therapist's working hours setting.
  • Strategy actions deep-link to existing flows where possible: outreach → Outreach tab, cluster → suggest message draft, block time → existing block flow.
  • Rendered in SmartBookingRail.jsx as a new TimeAvailableCard component, replacing RevenueCard.
